Unpooling Strategies — Section 5.1

Unpooling reverses spatial downsampling (e.g. after max pooling in an encoder). Three strategies differ in where they place the pooled value within each 2×2 block of the upsampled output.

Max-unpooling stores the exact position of the maximum value during the forward pooling pass, then restores it there during unpooling (all other cells = 0).  Bed-of-nails always places the pooled value at position (0,0) of each 2×2 block and zeros elsewhere — no index memory needed.  Nearest-neighbor fills all 4 cells of each 2×2 block with the pooled value, producing a smooth (but blocky) upsampling. Only max-unpooling requires storing extra information from the forward pass.
