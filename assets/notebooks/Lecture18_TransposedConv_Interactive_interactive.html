How transposed convolution works: For each input cell (r, c), we place a scaled copy of the kernel (input[r,c] × kernel) into the output at position r_start = r×stride − padding, c_start = c×stride − padding. Where scaled patches overlap, values are summed. This is why transposed convolution is also called a fractionally-strided convolution or deconvolution (informally). The kernel weights are learned, making this more powerful than fixed unpooling strategies. The kernel shown here is: 2×2 → [[1,2],[3,4]], 3×3 → [[1,0,1],[0,1,0],[1,0,1]].
